状态机暂时分为normal态、直道态、圆环态三者互斥。
状态机位置在原图边界提取后的角点检测和直边检测后面，目的是在原图上补线。
圆环相关阈值已迁到 `project/code/driver/vision/vision_config.c`，下面描述的是当前默认值。
normal->左圆环态的判断依据为，右边界检测到直边，左边界检测到角点，且右边界总点数大于20，同时左角点索引小于右边界总点数减20

进入左圆环态后，初始子状态为circle_1
circle_1:无操作
circle_1->circle_2:左边界从起始行就开始使用左边框作为边界，且连续使用15行以上的边框作为边界
circle_2:无操作
circle_2->circle_3:左边界的起始行不再使用左边框作为边界
circle_3:抛弃右边界所有巡到的线，观察左边界的规则数组（目标点选择使用规则数组，补线起点检测仍使用八邻域原始边界），此时左边界的规则数组会有一段连续的点（数量大于8）使用左边框作为边界，若找到，则使用这段边框边界结束后再往后8个索引点的坐标（若不存在就用最后的），与右边界起始行最右侧的点相连补线，补线是作为右边界使用的，左边界不动，经过规则化后送入ipm（这一段补线要在网页中以辅助线的形式渲染出来，方便看效果）
circle_3->circle_4:右边界从起始行开始，使用的边框边界超过20行
circle_4:无操作
circle_4->circle_5:右边界检测出角点
circle_5:观察左边界的规则数组（未裁减），此时左边界的规则数组会有一段连续的点（数量大于8）使用左边框作为边界，若找到，则使用这段边框边界结束的坐标，与右侧角点（若没检测到角点就用右边界起始行最右侧的点）相连补线，经过规则化后送入ipm（这一段补线要在网页中以辅助线的形式渲染出来，方便看效果）
circle_5->circle_6:右边界起始行使用右边框作为边界，且连续使用15行以上的边框作为边界
circle_6:搜线起始行提高到70
circle_6->normal:两边都检测到直边
