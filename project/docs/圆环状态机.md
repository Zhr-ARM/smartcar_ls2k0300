# 圆环状态机

## 1. 总体说明

当前视觉主状态暂分为四类，四者互斥：

- `normal`
- `直道态`
- `左圆环态`
- `右圆环态`

圆环状态机位于原图边界提取后的角点检测、直边检测之后，核心目的不是单纯识别圆环，而是基于当前边界结构在原图上执行补线，再将结果送入后续规则化与 IPM 流程。

圆环相关阈值已经迁移到 `project/code/driver/vision/vision_config.c`。本文记录的是当前默认规则与状态流转逻辑。

## 2. 进入条件

### 2.0 `normal -> 直道态`

满足以下条件时，从 `normal` 进入直道态：

1. 左边界未检测到角点。
2. 右边界未检测到角点。
3. 左边界检测到直边。
4. 右边界检测到直边。

直道态的作用是把“双侧稳定直边且无角点”的场景单独标识出来。

### 2.0.1 `直道态 -> normal`

直道态一旦不再满足以下任一条件，就立即退回 `normal`：

1. 两侧都没有角点。
2. 两侧都检测到直边。

也就是说：

1. 只要任一侧出现角点，退出直道态。
2. 只要任一侧不再是直边，退出直道态。
3. 退回 `normal` 后，再按 `normal` 的入口规则判断是否进入左右圆环等其他状态。

### 2.1 `normal -> 左圆环态`

满足以下条件时，从 `normal` 进入左圆环态：

1. 右边界检测到直边。
2. 左边界检测到角点。
3. 左角点的 `y` 坐标大于 `60`，即位于画面竖向下半区。
4. 右边界总点数大于 `20`。
5. 左角点索引小于 `右边界总点数 - 20`。
6. 从左角点 `y-10` 行开始，计算该行以及继续往上 `3` 行的原始左右边界 `x` 坐标差值（未补线、八邻域、未裁减）；若这些差值都大于 `80`，才满足条件。

进入左圆环态后，初始子状态为 `circle_1`。

### 2.2 `normal -> 右圆环态`

右圆环与左圆环按左右镜像处理，进入条件完全对称：

1. 左边界检测到直边。
2. 右边界检测到角点。
3. 右角点的 `y` 坐标大于 `60`，即位于画面竖向下半区。
4. 左边界总点数大于 `20`。
5. 右角点索引小于 `左边界总点数 - 20`。
6. 从右角点 `y-10` 行开始，计算该行以及继续往上 `3` 行的原始左右边界 `x` 坐标差值（未补线、八邻域、未裁减）；若这些差值都大于 `80`，才满足条件。

进入右圆环态后，初始子状态为 `circle_1`，后续子状态流转按左右镜像执行。

## 3. 子状态流转

### 3.1 `circle_1`

状态动作：

- 无额外操作。

转移条件：

- 当左边界从起始行开始就使用左边框作为边界，且连续使用左边框超过 `15` 行时，转入 `circle_2`。

### 3.2 `circle_2`

状态动作：

- 无额外操作。

转移条件：

- 当左边界起始行不再使用左边框作为边界时，转入 `circle_3`。

### 3.3 `circle_3`

状态动作：

1. 抛弃右边界当前巡到的全部线。
2. 观察左边界的规则数组。
3. 左圆环在该状态下将中线选择偏好切换为左边界优先；右圆环按左右镜像切换为右边界优先。
4. 补线起点固定使用右边界起始行最右侧的点。
5. 在左边界规则数组中寻找目标点，优先取贴左边框连续段结束后、后续分离直线段的首点，并尽量向左角点方向靠近。
6. 若未稳定识别出该分离直线段，则退化使用左边框连续段结束后再向后偏移的规则边界点。
7. 从右边固定起点向左边目标点连线，生成新的右边界数组；数组顺序保持从右边固定起点开始，符合边界传入 IPM 的顺序语义。
8. 该补线作为右边界使用，左边界保持不动。
9. 规则化后送入 IPM。

实现要求：

- 这一段补线需要在网页端以辅助线形式渲染，方便观察补线效果。

转移条件：

- 当右边界从起始行开始使用边框边界，且连续超过 `20` 行时，转入 `circle_4`。

### 3.4 `circle_4`

状态动作：

- 左圆环在该状态下将中线选择偏好切换为左边界优先；右圆环按左右镜像切换为右边界优先。

转移条件：

- 当右边界检测出角点时，转入 `circle_5`。

### 3.5 `circle_5`

状态动作：

1. 左圆环在该状态下将中线选择偏好切换为左边界优先；右圆环按左右镜像切换为右边界优先。
2. 左圆环补线时，起点使用右边界上的选点：优先取右侧角点；若未检测到右侧角点，则退化使用右边界起始行最右侧的点。
3. 左圆环补线目标点改为左边界规则数组中“边框边界连续段”的最后一个点。
4. 从右边起点向左侧边框连续段最后一点连线，生成新的右边界数组；数组顺序保持从右边起点开始，符合右边界传入 IPM 的顺序语义。
5. 右圆环按左右镜像处理：起点使用左边界上的选点，目标点改为右边界规则数组中“边框边界连续段”的最后一个点。
6. 将补线结果经过规则化后送入 IPM。

实现要求：

- 这一段补线同样需要在网页端以辅助线形式渲染。

转移条件：

- 当右边界起始行使用右边框作为边界，且连续使用右边框超过 `15` 行时，转入 `circle_6`。

### 3.6 `circle_6`

状态动作：

- 搜线起始行提高到配置项 `route_circle_stage6_maze_start_row` 指定的行号，当前默认值为 `70`。

转移条件：

- 当左右两侧都重新检测到直边时，退出圆环态并回到 `normal`。

## 4. 补线策略汇总

圆环阶段当前有两次明确的补线动作：

1. `circle_3`
   切换中线偏好到圆环内侧边界优先，并使用右边界起始行最右点作为固定起点，连向左侧规则边界中识别出的目标点，生成右边界补线。
2. `circle_5`
   使用本侧边界选点作为起点，连向对侧规则边界中边框连续段的最后一个点，生成本侧边界补线；同时切换中线偏好到圆环内侧边界优先。

共同规则：

- 补线只替代右边界使用。
- 左边界本身不改动。
- 补线结果需要参与后续规则化和 IPM。
- 补线辅助线需要对网页端可视化输出。

## 5. 实现备注

1. 本文描述的是左圆环流程，右圆环应按左右镜像方式实现。
2. 文中的行数阈值、连续点数量阈值、索引偏移量均以当前默认配置为准。
3. 若配置项后续继续迁移或改名，应优先以 `vision_config.c` 中实际值为准，并同步更新本文。
