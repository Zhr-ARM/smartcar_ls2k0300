# 当前代码视觉处理概述（按当前实现）

```text
/dev/video0
 -> vision_frame_capture.cpp 采集 full BGR
 -> vision_pipeline_process_step()
    -> vision_image_processor_process_step()
       -> full BGR -> 160x120 bgr/gray/rgb565
       -> OTSU -> binary_u8
       -> trace -> 边界/角点/直边/贴边行数
       -> route_state_machine_update
       -> CROSS/CIRCLE 子状态边界修正
       -> IPM(280x140) 与 line_error
    -> vision_infer_async（可选异步）
       -> 红框/ROI 回写 + 图像叠加
 -> vision_pipeline_send_step()
    -> vision_transport_send_step()
```

## 1. 主线程与分工

1. vision_thread
每轮先调用 vision_pipeline_process_step，再调用 vision_pipeline_send_step。

2. vision_image_processor
主巡线链核心：图像预处理、边界提取、状态机输入装配、边界修正、IPM、中线和 line_error。

3. vision_infer_async
异步辅助链，默认可关闭，不阻塞主巡线。

4. line_follow_thread
读取 line_error、track_point、route state，进行并级控制并下发左右轮目标。

## 2. 分辨率与核心缓存

1. full 图
UVC_WIDTH x UVC_HEIGHT（相机原始 BGR）

2. proc 图
160x120（bgr、gray、rgb565、binary_u8）

3. IPM 图
280x140

## 3. 每帧关键流程

1. 采图 + 可选去畸变
2. bgr/gray/rgb565 生成
3. OTSU 二值化
4. 左右边界 trace（当前配置 maze_trace_method=1，默认八邻域）
5. 十字下角点、起始贴边、直边等状态量提取
6. route_state_machine 更新（含 CROSS/STRAIGHT/CIRCLE）
7. 按状态对子边界进行修正：
   - CROSS_1/2：辅助边界与上角点截断逻辑
   - CIRCLE state3/state5：单侧补线
8. IPM 边界、中线、track point、line_error

## 4. 当前状态机在主链中的真实地位

当前五个主状态全部生效：

1. NORMAL
2. STRAIGHT
3. CIRCLE_LEFT
4. CIRCLE_RIGHT
5. CROSS

当前配置中：

1. route_cross_detection_enabled = true
2. route_circle_detection_enabled = true

NORMAL 下优先级：CROSS -> STRAIGHT -> CIRCLE_LEFT -> CIRCLE_RIGHT。

## 5. 推理链行为

vision_pipeline_process_step 的推理分支：

1. 若 infer 关闭或图像无效，清空推理状态并直接返回
2. 若 infer 开启，提交当前帧给异步线程
3. 读取最近完成结果并回填 red_rect/roi
4. 可按推理类别动态改 ipm_center_target_offset_from_left_px

所以推理链是“附加增强”，不决定主巡线链是否继续。

## 6. 外发与显示

1. vision_transport
负责逐飞助手和网页 UDP/TCP 输出，已包含 route_main_state、route_sub_state、preferred_source、cross/circle 调试字段。

2. screen_display_thread
按 send_mode 显示 binary/gray/rgb565 及叠加边界与控制信息。

## 7. 当前默认配置关键点

来源 [project/code/driver/vision/vision_config.c](project/code/driver/vision/vision_config.c#L347)：

1. route_cross_exit_start_gap_x_max = 155
2. route_circle_stage6_maze_start_row = 60
3. cross_aux_vertical_scan_max_rows = 75
4. cross_aux_trace_max_points = 50
5. cross_aux_trace_upward_rows_max = 30
6. route_straight_enter_consecutive_frames = 3
7. route_straight_abs_error_sum_max = 50.0f

## 8. 当前版本注意点

1. 十字链路已接入主状态机，并在 image_processor 中执行辅助边界与上角点处理。
2. CROSS_2 -> NORMAL 当前仅由起始边界间距阈值判断。
3. line_follow 已支持 NORMAL/STRAIGHT/CIRCLE/CROSS 四类参数包切换。
