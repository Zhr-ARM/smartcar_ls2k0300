## 2. 总体数据流（全状态共用骨架）

```text
图像输入(BGR/Gray/Binary)
  -> 原图追踪层(raw trace + dirs)
  -> 按行规则层(regular boundary)
  -> 状态信号层(route_input)
  -> route_state_machine_update
  -> 状态应用层(唯一改 src 边界)
  -> src 边界最终输出(left_pts/right_pts)
  -> IPM输入复制(left_ipm_pts/right_ipm_pts)
  -> IPM映射与中线生成
  -> line_error / track_point / transport调试输出
```

关键约束：

1. `left_pts/right_pts` 是 src 侧唯一真值边界。
2. `left_ipm_pts/right_ipm_pts` 始终由最终 src 边界复制得到。
3. 状态只影响“src 应用层怎么改边界”，不直接改 IPM 入参结构。

## 3. 分层数据对象与职责

## 3.1 原图追踪层（只读）

核心数据：

1. `left_trace_pts/right_trace_pts`
2. `left_trace_dirs/right_trace_dirs`
3. `left_trace_pts_raw/right_trace_pts_raw`
4. `left_trace_first_touch_y_work/right_trace_first_touch_y_work`
5. `left_start_frame_wall_rows/right_start_frame_wall_rows`

职责：

1. 从二值图提取边界原始轮廓与方向序列。
2. 提供角点、贴边、直边等信号原料。

## 3.2 按行规则层（只读）

核心数据：

1. `left_regular_pts/right_regular_pts`
2. `left_regular_num/right_regular_num`

职责：

1. 提供“一行一点”的规则边界基底。
2. 给状态应用层作为可控输入。

## 3.3 状态信号层（只读）

核心数据（经 `route_input` 汇总）：

1. 角点：`left/right_corner_found,x,y,index`
2. 贴边：`left/right_start_frame_wall_rows`
3. 直边：`left/right_straight`
4. 入口约束：`left/right_boundary_count`
5. 圆环入口 raw gap：`left/right_circle_entry_raw_gap_ok`
6. cross 退出相关：`start_boundary_gap_x`

## 3.4 状态应用层（唯一改数组）

核心数据：

1. `left_pts/right_pts`
2. `left_num/right_num`

职责：

1. 按主状态/子状态改写 src 边界。
2. 完成后一次性进入 IPM 输入复制。

## 3.5 IPM层（只消费应用层结果）

核心数据：

1. `left_ipm_pts/right_ipm_pts`
2. `g_ipm_xy_*`、`g_ipm_shift_*`

职责：

1. src->ipm 变换。
2. 后处理与中线平移。
3. 输出 `line_error`。

## 4. 各状态分支的数据走向

## 4.1 NORMAL / STRAIGHT

```text
raw trace触边检测
  -> touch_y_work
  -> truncate_regular_boundary_at_touch_y_inplace
  -> 回写 left_pts/right_pts
  -> copy 到 left_ipm_pts/right_ipm_pts
  -> IPM
```

说明：

1. 仅做触边截断类动作，不做 guide 替换。
2. `STRAIGHT` 与 `NORMAL` 在边界应用层动作一致，主要区别在状态机判定与控制参数档位。

## 4.2 CIRCLE（左右镜像）

### CIRCLE state3

```text
本侧 regular 边界
  -> find_circle_stage3_target_on_regular_boundary
  -> build_line_points_between(start, target)
  -> apply_circle_guide_replace_boundary(替换对侧边界)
  -> 圆环硬截断(按 route_circle_apply_touch_margin_px)
  -> IPM
```

### CIRCLE state5

```text
对侧角点(或起点) + 本侧regular目标
  -> find_circle_stage5_target_on_regular_boundary(..., circle_guide_anchor_offset_stage5, ...)
  -> build_line_points_between(start, target)
  -> apply_circle_guide_replace_boundary(替换对侧边界)
  -> 圆环硬截断(按 route_circle_apply_touch_margin_px)
  -> IPM
```

### CIRCLE 其它状态

1. `state1/2/4/6` 不做 guide 替换。
2. `state6` 额外影响 `maze_start_row` 下限（搜线起始行抬高）。

## 4.3 CROSS（CROSS_1 / CROSS_2 / CROSS_3）

### CROSS_1

```text
下角点命中?
  -> 否: 该侧边界清零
  -> 是: 下角点截断得到 cross_base
        + aux边界 + 上角点搜索(dir=5第N个或最后一个)
        -> 命中: base + bridge + aux_tail 回写
        -> 失败: 回退 cross_base
  -> IPM
```

### CROSS_2

```text
冻结的下角点
  -> 重建aux
  -> 重找上角点
  -> 命中: 输出上角点后段
  -> 失败: 该侧置空
  -> IPM
```

### CROSS_3（本次已参数化）

```text
regular边界
  -> build_cross_stage3_anchor_cut_boundary(
       anchor_x/y,
       jump_x_threshold,
       cut_forward_points)
  -> bridge + tail 回写
  -> IPM
```

## 6. 参数到行为的映射（优化后）

## 6.1 CROSS

1. `route_cross_stage1_enter_corner_y_min`  
   控制 `CROSS_1 -> CROSS_2` 的角点 y 触发阈值（状态机 + transport ready）。
2. `route_cross_stage3_jump_x_threshold_px`  
   控制 `CROSS_3` 跳变检测灵敏度。
3. `route_cross_stage3_cut_forward_points`  
   控制 `CROSS_3` 命中后切点前移程度。
4. `route_cross_stage3_left/right_anchor_x/y`  
   控制 `CROSS_3` 左右桥接锚点位置。

## 6.2 CIRCLE

1. `route_circle_entry_corner_y_min`  
   控制圆环入口角点 y 条件。
2. `circle_guide_anchor_offset_stage5`  
   控制 state5 目标锚点偏移（现已生效）。
3. `route_circle_apply_touch_margin_px`  
   控制圆环送 IPM 前触边截断边距。
