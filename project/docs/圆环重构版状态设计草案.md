# 圆环重构版状态设计草案（目标行为 + 简化转移图 + 参数清单）

## 0. 适用范围（本草案）

- 只覆盖圆环部分（`CIRCLE_LEFT / CIRCLE_RIGHT` 及其子状态）。
- 以当前代码为基线：
  - `project/code/driver/vision/vision_route_state_machine.cpp`
  - `project/code/driver/vision/vision_image_processor.cpp`
  - `project/user/smartcar_config.toml`
- 十字当前关闭（`route_cross_detection_enabled = false`），本草案不引入十字逻辑。
- 与你已调整的 `NORMAL` 架构对齐：识别层只读，应用层改写，最终统一送 IPM。

---

## 1. 目标行为（重构后）

1. 所有“状态判断信号”都在工作层产出，不改数组。
2. 圆环状态机只消费信号，不直接读写边界数组。
3. 圆环几何动作（裁剪、补线、替换）只在“按行规则数组应用层”执行，统一改 `left_apply/right_apply`。
4. IPM 输入只来自应用层最终数组，保持单轨真值链路。
5. 左右圆环逻辑完全镜像，用统一 `self/opp` 命名，减少左右分叉变量。

建议的主链：

`八邻域 raw 工作层（只读）`
-> `按行规则 regular 工作层（只读）`
-> `circle_detect_signals（只读聚合）`
-> `circle_state_machine_update（只读）`
-> `circle_apply_plan_build（产出动作计划）`
-> `circle_apply_execute（改 regular_apply 数组）`
-> `src final boundary`
-> `IPM`

---

## 2. 分层与数据契约（统一变量）

### 2.1 识别工作层（只读，不改数组）

输出统一信号 `circle_detect_signals_t`（建议）：

- `self_straight / opp_straight`
- `self_corner_found / opp_corner_found`
- `self_corner_x / self_corner_y / self_corner_index`
- `self_start_frame_wall_rows / opp_start_frame_wall_rows`
- `self_boundary_count_regular / opp_boundary_count_regular`
- `self_circle_entry_raw_gap_ok`
- `both_straight`

数据来源与当前实现对齐：

- 角点、判直、起始贴边：八邻域工作层缓存。
- `*_circle_entry_raw_gap_ok`：raw 边界行间距检查。
- `*_boundary_count_regular`：按行规则边界点数（当前 `left_num/right_num` 语义）。

### 2.2 状态机层（只读）

- 输入：`circle_detect_signals_t`。
- 输出：`main_state/sub_state/preferred_source`。
- 禁止状态机内做任何边界改写。

### 2.3 应用层（唯一改数组层）

输入：

- `left_regular_work[] / right_regular_work[]`
- `circle_state_snapshot`
- `circle_detect_signals_t`

输出：

- `left_apply[] / right_apply[]`（最终送 IPM）
- 可选调试缓存（guide 线、触边 y、动作标签）

## 3. 简化转移图（左右合并表达）

约定：`SIDE ∈ {LEFT, RIGHT}`，`SELF=SIDE`，`OPP=另一侧`。

```text
NORMAL / STRAIGHT
  -- circle_entry_ready(SIDE) -->
CIRCLE_SIDE_1
  -- self_start_frame_wall_rows >= stage_enter -->
CIRCLE_SIDE_2
  -- self_start_frame_wall_rows <= 0 -->
CIRCLE_SIDE_3
  -- opp_start_frame_wall_rows > stage3_trigger -->
CIRCLE_SIDE_4
  -- opp_corner_found -->
CIRCLE_SIDE_5
  -- opp_start_frame_wall_rows >= stage_enter -->
CIRCLE_SIDE_6
  -- both_straight -->
NORMAL
```

入口条件（`circle_entry_ready(SIDE)`）：

1. `opp_straight == true`
2. `self_corner_found == true`
3. `self_corner_y > 60`（建议参数化，默认 60）
4. `self_corner_index >= 0`
5. `opp_boundary_count_regular > route_circle_entry_min_boundary_count`
6. `self_circle_entry_raw_gap_ok == true`
7. `self_corner_index < opp_boundary_count_regular - route_circle_entry_corner_tail_margin`

## 4. 圆环应用层动作表（仅应用层改数组）

### 4.1 state3 / state5 guide 替换（保留现行为）

- `CIRCLE_LEFT_3`：用左 regular 找目标点，生成 guide，替换右 apply。
- `CIRCLE_LEFT_5`：从右角点或右起点到目标点，生成 guide，替换右 apply。
- `CIRCLE_RIGHT_3/5`：左右镜像。

### 4.2 圆环统一硬截断（送 IPM 前）

1. 去起始人工边框前缀。
2. 找首次“触边”（当前 margin=2px）。
3. 从首次触边处截断，不保留后续段。

### 4.3 state6 的搜线起始行抬高

- 保持现行为：`maze_start_row = max(maze_start_row, route_circle_stage6_maze_start_row)`。
- 该动作属于“预处理策略”，不改边界数组，但由圆环状态驱动。

## 5. 参数清单（圆环）

| 参数 | 当前值（smartcar_config.toml） | 层级 | 说明 |
|---|---:|---|---|
| `route_circle_detection_enabled` | `true` | 状态机 | 圆环总开关 |
| `route_circle_entry_min_boundary_count` | `20` | 识别层 | 入口对侧最小点数 |
| `route_circle_entry_corner_tail_margin` | `20` | 识别层 | 角点到对侧尾部保留余量 |
| `route_circle_entry_corner_row_offset` | `0` | 识别层 | raw gap 起查行偏移 |
| `route_circle_entry_gap_check_rows` | `0` | 识别层 | raw gap 检查行数（代码中最少按 1 行处理） |
| `route_circle_entry_min_raw_boundary_gap` | `0` | 识别层 | raw gap 最小间距 |
| `route_circle_stage_frame_wall_rows_enter` | `30` | 状态机 | state1/5 进下一阶段阈值 |
| `route_circle_stage3_frame_wall_rows_trigger` | `70` | 状态机 | state3->4 对侧贴边阈值 |
| `route_circle_stage6_maze_start_row` | `50` | 预处理策略 | state6 抬高搜线起始行 |
| `circle_guide_min_frame_wall_segment_len` | `8` | 应用层 | guide 目标段最小贴边长度 |
| `circle_guide_target_offset_stage3` | `5` | 应用层 | state3 目标后移偏移 |
| `circle_guide_anchor_offset_stage5` | `0` | 应用层 | 配置已存在，当前主链未实际使用 |
| `route_circle_center_target_offset_from_left_px` | `14.0` | IPM/控制联动 | circle4/5 中线目标偏移（右圆镜像） |

建议补齐为显式参数（当前为硬编码）：

- `route_circle_entry_corner_y_min`（默认 `60`）
- `route_circle_apply_touch_margin_px`（默认 `2`）

## 6. 兼容命名映射（旧变量 -> 新统一语义）

- `left/right_start_frame_wall_rows` -> `self/opp_start_frame_wall_rows`
- `left/right_corner_found` -> `self/opp_corner_found`
- `left/right_boundary_count` -> `self/opp_boundary_count_regular`
- `left/right_circle_entry_raw_gap_ok` -> `self_circle_entry_raw_gap_ok`
- `left_pts/right_pts`（状态后直接改） -> `left/right_regular_work` + `left/right_apply`

实现建议：先做“别名字段 + 适配层”，不改现有配置键名，保证兼容。

---

## 7. 最小落地步骤（仅圆环）

1. 抽出 `circle_detect_signals_t`，把圆环入口/推进信号统一从工作层填充。
2. 状态机改为只吃 `signals`，保持现有状态编号与转移条件不变。
3. 抽出 `circle_apply_execute()`，集中 state3/state5 guide 与 circle 硬截断。
4. 把 IPM 输入固定为 `apply` 后数组，去掉状态分散改写入口。
5. 最后再把 `60`、`2px` 变成配置项（默认值保持现行为）。
