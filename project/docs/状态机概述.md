# 状态机概述

## 1. 范围

当前仓库里“真正接入视觉主链并影响控制”的状态机，是 `vision_route_state_machine`。

它位于：

`原图边界/角点/直边检测 -> route_state_machine -> 圆环补线/IPM/控制参数选择`

所以它不是单纯的“赛道标签器”，而是同时决定：

1. 是否进入直道态或圆环态
2. 圆环补线是否生效
3. IPM 中线优先来源
4. 控制线程使用哪套参数包

## 2. 当前状态集合

### 2.1 主状态

当前枚举包含：

1. `NORMAL`
2. `STRAIGHT`
3. `CIRCLE_LEFT`
4. `CIRCLE_RIGHT`
5. `CROSS`

当前实际生效的是：

1. `NORMAL`
2. `STRAIGHT`
3. `CIRCLE_LEFT`
4. `CIRCLE_RIGHT`

`CROSS` 当前仅保留枚举和子状态名，默认配置 `route_cross_detection_enabled = false`，主链也没有进入它的状态转移。

### 2.2 子状态

当前子状态包含：

1. `NONE`
2. `CROSS_BEGIN`
3. `CROSS_IN`
4. `CIRCLE_LEFT_1 ~ CIRCLE_LEFT_6`
5. `CIRCLE_RIGHT_1 ~ CIRCLE_RIGHT_6`

其中当前真正会跑到的是：

1. `NONE`
2. `CIRCLE_LEFT_1 ~ CIRCLE_LEFT_6`
3. `CIRCLE_RIGHT_1 ~ CIRCLE_RIGHT_6`

## 3. 输入、输出与调用时机

### 3.1 输入

状态机每帧由 `vision_image_processor_process_step()` 组装 `vision_route_state_input_t` 后调用。

主要输入包括：

1. 左右角点是否存在、坐标、索引
2. 左右原图直边状态
3. 左右起始贴边连续行数
4. 左右边界点数
5. 圆环入口原始边界 gap 检查结果
6. 基础 `preferred_source`
7. 本帧编码器增量

### 3.2 输出

状态机对外暴露：

1. `main_state`
2. `sub_state`
3. `preferred_source`
4. `encoder_since_state_enter`
5. `cross_loss_count`
6. `left_loss_count / left_gain_count`
7. `right_loss_count / right_gain_count`

当前只有前四类真正被主链使用得更明显：

1. `main_state/sub_state` 被控制线程读走，用来切参数包
2. `preferred_source` 被 IPM 处理中线选择逻辑使用
3. `encoder_since_state_enter` 主要用于网页端观测

后四个计数器当前会清零和上报，但还没有完整业务逻辑去驱动它们。

## 4. 状态流转总览

当前主状态逻辑可以简化成：

```text
NORMAL
----> STRAIGHT
-------> NORMAL

NORMAL
----> CIRCLE_LEFT -> CIRCLE_LEFT_1 -> 2 -> 3 -> 4 -> 5 -> 6 -> NORMAL

NORMAL
----> CIRCLE_RIGHT -> CIRCLE_RIGHT_1 -> 2 -> 3 -> 4 -> 5 -> 6 -> NORMAL
```

### 4.1 `NORMAL`

默认状态。

行为：

1. `preferred_source` 跟随基础配置 `ipm_line_error_source`
2. 若双侧直边且双侧无角点，转入 `STRAIGHT`
3. 若满足左/右圆环入口，转入对应圆环态

### 4.2 `STRAIGHT`

直道稳定态。

行为：

1. 仍使用基础 `preferred_source`
2. 只要“双侧直边且双侧无角点”破坏，就回 `NORMAL`

### 4.3 `CIRCLE_LEFT / CIRCLE_RIGHT`

左右圆环态。

行为：

1. 主状态内部再细分 6 个子状态
2. `state3` 和 `state5` 会触发圆环补线
3. 控制层会切换到圆环专用参数包

## 5. 和视觉处理链的连接点

状态机处在视觉处理链中段，前后连接关系如下：

1. 前级提供原图边界、角点、直边、贴边连续行数
2. 状态机输出 `main_state/sub_state/preferred_source`
3. `vision_image_processor` 根据子状态决定是否补线
4. IPM 模块根据 `preferred_source` 决定优先选左平移中线还是右平移中线
5. `vision_line_error_layer` 基于选中的 IPM 中线计算 `line_error`
6. `line_follow_thread` 根据状态切换 `NORMAL / STRAIGHT / CIRCLE_ENTER / CIRCLE_INSIDE / CIRCLE_EXIT`

这说明当前状态机是视觉链和控制链之间的桥。

## 6. 和控制层的连接点

`line_follow_thread.cpp` 中当前参数选择关系为：

1. `STRAIGHT` -> `kStraightProfile`
2. `CIRCLE_*_1` -> `kCircleEnterProfile`
3. `CIRCLE_*_2/3/4` -> `kCircleInsideProfile`
4. `CIRCLE_*_5/6` -> `kCircleExitProfile`
5. 其他状态 -> `kNormalProfile`

因此状态机不仅告诉控制“现在是什么路”，还会改变：

1. 基础速度
2. 位置环参数
3. 角速度环参数
4. 转向限幅和降速策略

## 7. 当前现状总结

如果只看当前代码而不看历史方案，可以把它理解为：

1. 已落地的是“直道 + 左右圆环”的轻量路由状态机。
2. 十字相关枚举保留，但还没有进入主流程。
3. 圆环状态机已经和补线、IPM 选边、控制参数切换三条链路绑定。
4. 现在最值得维护的文档重点，不是“未来可能有哪些状态”，而是“当前哪些状态真的在驱动视觉和控制”。
