# 十字状态机（按当前代码）

## 1. 实现位置

1. 状态转移： [project/code/driver/vision/vision_route_state_machine.cpp](project/code/driver/vision/vision_route_state_machine.cpp)
2. 辅助线与上角点处理： [project/code/driver/vision/vision_image_processor.cpp](project/code/driver/vision/vision_image_processor.cpp)
3. 调试字段输出： [project/code/driver/vision/vision_transport.cpp](project/code/driver/vision/vision_transport.cpp)

在 NORMAL 中，十字入口优先级高于直道和圆环。

## 2. 状态定义

1. 主状态：CROSS
2. 子状态：CROSS_1、CROSS_2

## 3. 进入条件（NORMAL/STRAIGHT -> CROSS_1）

由 cross_entry_ready 判断：

1. route_cross_detection_enabled = true
2. left_corner_found && right_corner_found
3. left_corner_index >= 0 && right_corner_index >= 0
4. left_corner_post_frame_wall_rows >= route_cross_entry_corner_post_frame_wall_rows_min
5. right_corner_post_frame_wall_rows >= route_cross_entry_corner_post_frame_wall_rows_min

当前配置值：

1. route_cross_entry_corner_post_frame_wall_rows_min = 5

## 4. CROSS_1 逻辑

每侧处理流程：

1. 从下角点沿同一 x 向上找白->黑过渡点
2. 以过渡点为起点做八邻域 trace
3. 抽取每行一个点形成 regular 边界
4. 成功后缓存 aux_found、aux_transition、aux_trace、aux_regular

关键函数：

1. build_cross_aux_boundary
2. find_vertical_white_to_black_transition
3. trace_boundary_eight_neighbor
4. extract_one_point_per_row_from_contour

参数：

1. cross_aux_vertical_scan_max_rows = 75
2. cross_aux_trace_max_points = 50
3. cross_aux_trace_upward_rows_max = 30

在 CROSS_1 内，上角点融合路径的执行前置是：

1. g_cross_left_aux_found 或 g_cross_right_aux_found 为 true
2. 对应 aux_regular_num > 0
3. 在 aux_trace_dir 中找到第 3 个 dir=5（find_nth_dir_point_on_trace）

## 5. CROSS_1 -> CROSS_2

当前代码条件是“左右起始贴边行数同时达到阈值”：

1. left_start_frame_wall_rows >= route_cross_stage2_enter_start_frame_wall_rows_min
2. right_start_frame_wall_rows >= route_cross_stage2_enter_start_frame_wall_rows_min

当前阈值：

1. route_cross_stage2_enter_start_frame_wall_rows_min = 1

## 6. CROSS_2 逻辑

在 CROSS_2 中，按原始 trace dir 模板找上角点：

1. 先找 4 平台 -> 过渡 -> 6 平台
2. 再从记录点后续找 dir=5 角点
3. 找到后将 regular 边界在上角点之前截断

参数：

1. cross_upper_dir4_pre_run_len = 3
2. cross_upper_transition_max_len = 3
3. cross_upper_dir6_post_run_len = 3

## 7. CROSS_2 -> NORMAL（已按最新代码更新）

当前仅由起始边界间距决定：

1. start_boundary_gap_x > 0
2. start_boundary_gap_x < route_cross_exit_start_gap_x_max

当前阈值：

1. route_cross_exit_start_gap_x_max = 155

不再要求左右起始行都不贴边。

## 8. 对外调试字段

transport 已输出：

1. cross_lower_left/right_corner_*
2. cross_left/right_aux_found
3. cross_left/right_aux_trace、cross_left/right_aux_trace_dir
4. cross_left/right_aux_regular
5. cross_left/right_upper_corner_*
6. route_main_state、route_sub_state、route_preferred_source

可直接用于网页侧确认每帧十字推进与退出条件是否命中。
