# 环岛状态机全流程（当前代码）

本文基于当前代码实现梳理环岛（`CIRCLE_LEFT / CIRCLE_RIGHT`）完整链路，覆盖：
- 状态机入口与转移
- 输入信号来源
- 各子状态对原图边界的处理动作
- 进入 IPM 后中线策略
- 可观测调试字段

## 1. 状态定义

主状态与子状态定义在 `vision_route_state_machine.h`：
- 主状态：
  - `VISION_ROUTE_MAIN_NORMAL = 0`
  - `VISION_ROUTE_MAIN_STRAIGHT = 1`
  - `VISION_ROUTE_MAIN_CIRCLE_LEFT = 2`
  - `VISION_ROUTE_MAIN_CIRCLE_RIGHT = 3`
  - `VISION_ROUTE_MAIN_CROSS = 4`
- 环岛子状态：
  - 左环：`VISION_ROUTE_SUB_CIRCLE_LEFT_1..6`（枚举值 4..9）
  - 右环：`VISION_ROUTE_SUB_CIRCLE_RIGHT_1..6`（枚举值 10..15）

## 2. 每帧总体执行顺序（环岛相关）

在 `vision_image_processor.cpp` 的主处理流程中，环岛相关顺序如下：
1. 读取上一帧状态快照 `route_snapshot_before_trace`。
2. 八邻域巡线，得到左右 trace 点、方向 dir、角点缓存、起始贴边行数等工作层信号。
3. 组装 `vision_route_state_input_t route_input`，包括环岛入口判定信号（`left/right_circle_entry_raw_gap_ok`）。
4. 调用 `vision_route_state_machine_update(&route_input)` 更新主/子状态。
5. 根据新状态执行边界应用层改写：
   - 环岛 `state3/state5` 做 guide 补线并替换对侧边界。
   - 环岛主状态下统一执行“首次触边硬截断”。
6. 将最终 src 边界送入 IPM，生成平移中线；在环岛 `state4/state5` 启用环岛专用中线偏移。

## 3. 状态机输入信号（环岛相关）

状态机输入结构体：`vision_route_state_input_t`。环岛核心字段：
- `left_straight` / `right_straight`
- `left_corner_found/right_corner_found`
- `left_corner_y/right_corner_y`
- `left_corner_index/right_corner_index`
- `left_boundary_count/right_boundary_count`
- `left_start_frame_wall_rows/right_start_frame_wall_rows`
- `left_circle_entry_raw_gap_ok/right_circle_entry_raw_gap_ok`

这些量的来源：
- `left/right_straight`：由前段 dir 分布判断直边（`detect_src_straight_boundary_from_dirs`）。
- `left/right_start_frame_wall_rows`：由 `count_leading_side_frame_wall_rows` 统计“边界起点连续贴边行数”。
- 角点字段：来自下角点检测缓存（cross/circle 共用角点信号源）。
- `left/right_circle_entry_raw_gap_ok`：`circle_entry_raw_boundary_gap_ready(...)` 在 raw 轮廓上按行检查左右间距。

## 4. 环岛入口判定

### 4.1 左环入口 `left_circle_entry_ready`

满足以下条件才可由 `NORMAL/STRAIGHT` 进入 `CIRCLE_LEFT_1`：
1. `right_straight == true`
2. `left_corner_found == true`
3. `left_corner_y > route_circle_entry_corner_y_min`
4. `left_corner_index >= 0`
5. `right_boundary_count > route_circle_entry_min_boundary_count`
6. `left_circle_entry_raw_gap_ok == true`
7. `left_corner_index < (right_boundary_count - route_circle_entry_corner_tail_margin)`

### 4.2 右环入口 `right_circle_entry_ready`

与左环镜像，左右互换。

### 4.3 入口优先级

在 `NORMAL/STRAIGHT` 中优先级是：
1. `CROSS`
2. `CIRCLE`
3. `STRAIGHT`

## 5. 环岛 6 阶段转移（当前实现）

### 5.1 左环 `CIRCLE_LEFT_1..6`

状态转移条件：
1. `LEFT_1 -> LEFT_2`：
   `left_start_frame_wall_rows >= route_circle_stage_frame_wall_rows_enter`
2. `LEFT_2 -> LEFT_3`：
   `left_start_frame_wall_rows <= 0`
3. `LEFT_3 -> LEFT_4`：
   `right_start_frame_wall_rows > route_circle_stage3_frame_wall_rows_trigger`
4. `LEFT_4 -> LEFT_5`：
   `right_corner_found == true`
5. `LEFT_5 -> LEFT_6`：
   `right_start_frame_wall_rows >= route_circle_stage_frame_wall_rows_enter`
6. `LEFT_6 -> NORMAL`：
   `left_straight && right_straight`

### 5.2 右环 `CIRCLE_RIGHT_1..6`

完全镜像（左/右互换）。

### 5.3 运行期开关行为

若 `route_circle_detection_enabled == false` 且当前主状态在 circle，状态机会立即回到 `NORMAL`。

## 6. 各子状态的边界应用动作

### 6.1 state3：生成 guide 并替换对侧边界

左环 `CIRCLE_LEFT_3`：
1. 起点 `fixed_start` 取右侧边界起点（优先 regular，其次 apply，最后兜底点）。
2. 在左 regular 边界上调用 `find_circle_stage3_target_on_regular_boundary(...)` 找目标点。
3. `build_line_points_between(fixed_start, target)` 生成 guide。
4. `apply_circle_guide_replace_boundary(...)` 用 guide 替换右边界。

右环 `CIRCLE_RIGHT_3` 镜像，替换左边界。

### 6.2 state5：从角点/起点连向目标并替换对侧边界

左环 `CIRCLE_LEFT_5`：
1. 起点 `start` 优先取右角点；若无角点，回落到右边界起点/兜底点。
2. 目标点由 `find_circle_stage5_target_on_regular_boundary(..., circle_guide_anchor_offset_stage5, ...)` 获取。
3. 连线生成 guide 并替换右边界。

右环 `CIRCLE_RIGHT_5` 镜像，替换左边界。

### 6.3 Circle 主状态统一硬截断（送 IPM 前）

只要主状态是 `CIRCLE_LEFT/RIGHT`，都会执行：
1. `trim_initial_artificial_frame_prefix_inplace(...)` 去起始人工边框前缀。
2. `find_first_frame_touch_index_with_margin(..., route_circle_apply_touch_margin_px)` 找首次触边索引。
3. 直接截断到首次触边之前（不保留触边点）。

## 7. 与 normal/straight 的关系（影响入口信号）

当前实现中，`NORMAL/STRAIGHT` 分支在触边截断后，会再次执行角点截断，保证两者同时生效。这会影响下一帧进入环岛的边界点分布与角点相关信号稳定性。

## 8. IPM 中线在环岛的特殊策略

IPM 处理链默认使用 `ipm_center_target_offset_from_left_px`。  
但在以下子状态会切到环岛专用偏移：
- 左环：`CIRCLE_LEFT_4`、`CIRCLE_LEFT_5`
- 右环：`CIRCLE_RIGHT_4`、`CIRCLE_RIGHT_5`（镜像偏移 `track_width - route_circle_center_target_offset_from_left_px`）

另：此前“IPM 底边中点补到中线起点”的补线逻辑已移除，当前不再补该段。

## 9. 参数清单（环岛核心）

结构定义在 `vision_config.h`，默认值在 `vision_config.c`，运行值以 `smartcar_config.toml` 为准。

关键参数：
- `route_circle_detection_enabled`
- `route_circle_entry_min_boundary_count`
- `route_circle_entry_corner_tail_margin`
- `route_circle_entry_corner_row_offset`
- `route_circle_entry_gap_check_rows`
- `route_circle_entry_min_raw_boundary_gap`
- `route_circle_entry_corner_y_min`
- `route_circle_stage_frame_wall_rows_enter`
- `route_circle_stage3_frame_wall_rows_trigger`
- `route_circle_stage6_maze_start_row`
- `route_circle_center_target_offset_from_left_px`
- `circle_guide_min_frame_wall_segment_len`
- `circle_guide_target_offset_stage3`
- `circle_guide_anchor_offset_stage5`
- `route_circle_apply_touch_margin_px`

当前用户配置（`project/user/smartcar_config.toml`）已开启环岛识别：
- `route_circle_detection_enabled = true`

## 10. 调试观测建议（现有可直接看）

`vision_transport.cpp` 已上报：
- 路由状态：
  - `route_main_state`
  - `route_sub_state`
  - `route_preferred_source`
  - `route_encoder_since_enter`
- 环岛阈值配置镜像：
  - `route_circle_*`
  - `circle_guide_*`
- 现场信号：
  - `left_start_frame_wall_rows`
  - `right_start_frame_wall_rows`
  - `left_circle_guide_line`
  - `right_circle_guide_line`

调试判读要点：
1. `route_sub_state` 是否按 `1->2->3->4->5->6`（或镜像）推进。
2. `state3/state5` 时对侧 `*_circle_guide_line` 是否出现并稳定。
3. `left/right_start_frame_wall_rows` 是否满足对应阶段阈值条件。
4. `state4/state5` 时中线偏移是否符合环岛专用偏移逻辑。

## 11. 备注

1. 环岛入口使用的角点信号与 cross 共用检测源，属于当前代码架构的既有设计。  
2. `route_left/right_loss_count` 等计数在当前状态机实现中已定义并上报，但未见有效递增逻辑，调试时可视为保留字段。
