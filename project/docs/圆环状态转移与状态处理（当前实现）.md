# 圆环状态转移与状态处理（当前实现）

本文按当前代码提取，主要对应以下文件：
- `project/code/driver/vision/vision_route_state_machine.h`
- `project/code/driver/vision/vision_route_state_machine.cpp`
- `project/code/driver/vision/vision_image_processor.cpp`
- `project/code/app/line_follow_thread/line_follow_thread.cpp`
- `project/user/smartcar_config.toml`

说明：本文只描述“当前实际生效逻辑”，不讨论历史版本。

---

## 1. 状态定义与输入来源

主状态定义（`vision_route_state_machine.h`）：
- `NORMAL`
- `STRAIGHT`
- `CIRCLE_LEFT`
- `CIRCLE_RIGHT`
- `CROSS`

圆环子状态定义：
- 左圆：`CIRCLE_LEFT_1 ~ CIRCLE_LEFT_6`
- 右圆：`CIRCLE_RIGHT_1 ~ CIRCLE_RIGHT_6`

状态机输入 `vision_route_state_input_t` 的关键字段及来源：
- `left/right_straight`：来自八邻域工作层方向序列判直结果。
- `left/right_corner_found`、`corner_index`、`corner_y`：来自八邻域工作层角点检测缓存。
- `left/right_start_frame_wall_rows`：来自八邻域工作层起始贴边行统计。
- `left/right_boundary_count`：来自当前帧最终原图边界点数（`left_num/right_num`，已过常规处理）。
- `left/right_circle_entry_raw_gap_ok`：来自八邻域原始轮廓（raw）行间距检查。

结论：圆环状态机是“多源输入”，但推进核心信号主要来自八邻域层；入口点数约束来自最终边界计数。

## 2. 入口阶段（NORMAL/STRAIGHT -> CIRCLE）

入口判断在 `vision_route_state_machine.cpp`：

左圆入口 `left_circle_entry_ready`：
- `right_straight == true`
- `left_corner_found == true`
- `left_corner_y > 60`
- `left_corner_index >= 0`
- `right_boundary_count > route_circle_entry_min_boundary_count`
- `left_circle_entry_raw_gap_ok == true`
- `left_corner_index < right_boundary_count - route_circle_entry_corner_tail_margin`

右圆入口 `right_circle_entry_ready`（左右镜像）：
- `left_straight == true`
- `right_corner_found == true`
- `right_corner_y > 60`
- `right_corner_index >= 0`
- `left_boundary_count > route_circle_entry_min_boundary_count`
- `right_circle_entry_raw_gap_ok == true`
- `right_corner_index < left_boundary_count - route_circle_entry_corner_tail_margin`

入口优先级（`NORMAL` 和 `STRAIGHT` 都一致）：
- 先尝试 `CROSS`
- 再尝试圆环（左后右）
- 再考虑 straight 维持/进入（`NORMAL` 内）

## 3. 圆环状态机子状态转移

### 3.1 左圆（CIRCLE_LEFT）

`LEFT_1 -> LEFT_2`
- 条件：`left_start_frame_wall_rows >= route_circle_stage_frame_wall_rows_enter`

`LEFT_2 -> LEFT_3`
- 条件：`left_start_frame_wall_rows <= 0`

`LEFT_3 -> LEFT_4`
- 条件：`right_start_frame_wall_rows > route_circle_stage3_frame_wall_rows_trigger`

`LEFT_4 -> LEFT_5`
- 条件：`right_corner_found == true`

`LEFT_5 -> LEFT_6`
- 条件：`right_start_frame_wall_rows >= route_circle_stage_frame_wall_rows_enter`

`LEFT_6 -> NORMAL`
- 条件：`left_straight && right_straight`

### 3.2 右圆（CIRCLE_RIGHT）

`RIGHT_1 -> RIGHT_2`
- 条件：`right_start_frame_wall_rows >= route_circle_stage_frame_wall_rows_enter`

`RIGHT_2 -> RIGHT_3`
- 条件：`right_start_frame_wall_rows <= 0`

`RIGHT_3 -> RIGHT_4`
- 条件：`left_start_frame_wall_rows > route_circle_stage3_frame_wall_rows_trigger`

`RIGHT_4 -> RIGHT_5`
- 条件：`left_corner_found == true`

`RIGHT_5 -> RIGHT_6`
- 条件：`left_start_frame_wall_rows >= route_circle_stage_frame_wall_rows_enter`

`RIGHT_6 -> NORMAL`
- 条件：`left_straight && right_straight`

## 4. 圆环状态下的图像侧处理

圆环状态影响不仅是“状态标签”，还会直接改边界。

### 4.1 迷宫起始行调整（state6）

在“进入本帧 trace 前”：
- 若上一帧是 `CIRCLE_LEFT_6` 或 `CIRCLE_RIGHT_6`，则把 `maze_start_row` 至少抬到 `route_circle_stage6_maze_start_row`。
- 目的是提高出环阶段搜线稳定性。

### 4.2 子状态专用 guide 补线（state3/state5）

左圆：
- `CIRCLE_LEFT_3`：根据左侧 regular 边界找目标点，生成 guide，替换右边界。
- `CIRCLE_LEFT_5`：从右角点或右侧起点连向目标点，生成 guide，替换右边界。

右圆（镜像）：
- `CIRCLE_RIGHT_3`：根据右侧 regular 边界找目标点，生成 guide，替换左边界。
- `CIRCLE_RIGHT_5`：从左角点或左侧起点连向目标点，生成 guide，替换左边界。

### 4.3 圆环送 IPM 前硬截断

当主状态是圆环时：
- 先删除起始人工边框前缀；
- 再找首次触边位置（圆环专用判定：距边框 2px 内即视为触边）；
- 从首次触边处截断，不保留后续段。

这段逻辑与 `NORMAL/STRAIGHT` 的触边处理分支是并行独立的。

## 5. `preferred_source` 与中线偏移

状态机默认偏好源：
- `CIRCLE_LEFT` 主状态默认 `RIGHT`
- `CIRCLE_RIGHT` 主状态默认 `LEFT`

但在 circle `state3/4/5` 内，状态机会临时把偏好切到对侧（左圆切 `LEFT`，右圆切 `RIGHT`）。

IPM 侧中线偏移在 circle `state4/5` 有专门偏置：
- 左圆 `state4/5`：`target_offset_from_left_px = route_circle_center_target_offset_from_left_px`
- 右圆 `state4/5`：采用镜像偏置 `track_width - left_offset`

---

## 6. 与 NORMAL/STRAIGHT 的关键差异

`NORMAL/STRAIGHT`：
- 当前分支主要执行“八邻域检测触边 -> 用 touch_y 截断 regular -> 回写最终边界”。
- 不做 circle guide 替换。

圆环主状态：
- 不走上述 normal 触边分支。
- 走圆环子状态 guide 替换（3/5）和圆环专用送 IPM 前硬截断。
- 在 `state6` 还会抬高迷宫起始行。

---

## 7. 控制层参数档位映射（line_follow）

`line_follow_thread` 对圆环子状态映射如下：
- `CIRCLE_*_1` -> `CIRCLE_ENTER`
- `CIRCLE_*_2/3/4` -> `CIRCLE_INSIDE`
- `CIRCLE_*_5/6` -> `CIRCLE_EXIT`

所以圆环子状态不仅影响视觉边界，也直接影响控制参数组选择。

## 9. 备注（便于后续重构）

如果你要重整圆环，建议先明确这三层职责边界：
- 状态检测层：哪些信号必须来自八邻域，哪些可来自 regular/final。
- 几何处理层：guide 替换和触边截断是否全部统一在 regular 层完成。
- 输出层：`left_pts/right_pts` 到 IPM 的唯一真值链路是否保持单轨。
